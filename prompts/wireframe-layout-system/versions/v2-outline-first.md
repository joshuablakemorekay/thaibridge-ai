# v2 — content-first ("plan on paper")

```
Start with the outline so the plan is on paper first
```

**What changed:** instead of jumping into styling, I stepped back to plan the content first — a written site outline of every page and what it's *for* — before choosing any layout. This is the "create an outline" step from the web-design method, and it's what turned a vague "add a wireframe" into a deliberate per-page plan.
